Which client conditions actually bind the numbers?

A client's tactical instructions vary endlessly, but their shape falls into just three handling tiers. Rather than write a rule per condition, the copilot classifies each ask and is honest about what it can enforce today — so nobody forwards a proposal assuming an instruction is binding when it is only a note.

🔒 Enforced
Binds a computed number

Checked against real data and allowed to gate a trade — a target weight (applied to the sleeves), or an absolute price level on an instrument we can price.

“buy gold below $4,000/oz”
📡 Monitored
Watched, not yet binding

Placed on the watchlist and flagged when hit, but can't gate the math yet — typically a relative move that needs a defined reference point.

“add after a 15–20% pullback”
📝 Advisory
Shapes the write-up only

Folds into the narrative and the deck notes — execution style, selection preferences, open questions. Never touches a figure.

How wide is the door, really?

An honest read of the payoff

  • Through the enforcement door today, exactly one of these conditions (gold < $4,000) actually binds a number.
  • One more (the pullback) binds as soon as we define its reference point — cheap.
  • Everything else — tranches, low-fees, rate views — stays 📡/📝 no matter what: either a preference/viewpoint that is never a numeric gate, or blocked on data we don't yet hold (a fee/liquidity database).

So the enforcement handler has a narrow immediate payoff — but the tier labelling pays off across all six. The value isn't that gold gets gated; it's that the deck stops silently pretending the other five are handled. We sell transparency about what's binding, not a promise to enforce everything.

The guardrail — now tiered, not absolute

The copilot still never invents a number. An enforced condition can gate or flag a trade — using a figure sourced with provenance — but it can never fabricate one. Advisory and monitored items shape the words and the watchlist, never the maths.
